# Open Workout App

A small, self-hosted, mobile-first web app for logging exercises, activities,
and workouts. Everything is stored locally in a SQLite database — no
accounts, no cloud sync, no analytics.

**Version:** 0.1.7

## Features

- **Exercises** you define yourself, each configured with the fields it
  actually needs: weight (kg), duration (minutes), and/or reps — any
  combination, including all three at once (e.g. a weighted, timed carry).
  Exercises can also have a description.
- **Activities** are individual logged sets. The "new activity" screen shows
  your 10 most recently used exercises first, then filters as you type. New
  entries are pre-filled with your last values for that exercise so repeat
  logging is fast. Activities can be edited or deleted after the fact and
  can carry their own free-text note.
- **Workouts** automatically group activities logged on the same day, with a
  readable label like `wed 9-7-2026`, and each workout has its own editable
  comment field.
- **Exercise history** for each exercise shows a small chart of weight (or
  duration, if the exercise doesn't track weight) over time, plus the full
  list of past activities.
- **CSV export/import** on the Settings page backs up and restores all
  exercises and activities in one file.

## Technical background

- **Backend:** Python, [Flask](https://flask.palletsprojects.com/), organized
  as blueprints per resource (exercises, activities, workouts, settings).
- **Data:** [SQLAlchemy](https://www.sqlalchemy.org/) ORM over a SQLite file.
- **Templates:** [Jinja2](https://jinja.palletsprojects.com/), server-rendered.
- **Frontend:** [Pico CSS](https://picocss.com/) for styling and
  [htmx](https://htmx.org/) for the dynamic bits (exercise search, inline
  comment saving) — no build step, no JavaScript framework, no bundler.
- **Charts:** a small hand-rolled inline SVG generator (`app/charts.py`), no
  charting library.

## Installation

### Docker (recommended)

Using `docker-compose`:

```bash
git clone https://github.com/nimdaz/open-workout-app.git
cd open-workout-app
docker compose up -d
```

The app will be available at `http://localhost:5000`. Data is stored in
`./data` on the host, so it survives container restarts and rebuilds.

Without compose:

```bash
docker build -t open-workout-app .
docker run -d -p 5000:5000 -v "$(pwd)/data:/code/data" open-workout-app
```

### Running locally with a virtual environment

Requires Python 3.10+.

```bash
git clone https://github.com/nimdaz/open-workout-app.git
cd open-workout-app
python -m venv .venv
source .venv/bin/activate    # on Windows: .venv\Scripts\activate
pip install -r requirements.txt
python run.py
```

Then open `http://localhost:5000`. By default the SQLite database is created
at `app/data/app.db`; set the `DATA_DIR` environment variable to change that.

> The built-in server (`python run.py`) is a development server. If you're
> exposing this beyond your own machine or home network, put it behind a
> proper WSGI server (e.g. gunicorn) and a reverse proxy.

## A note on how this was built

This app was built collaboratively with **Claude Sonnet 5** (Anthropic),
used interactively through a **free Claude.ai account**. Development was
iterative: each feature was specified in detail, generated by the model,
and then manually reviewed and tested (not just skimmed) before moving on
to the next one. So this is AI-assisted development guided step-by-step by
a human, not autonomous or "vibe-coded" generation — but it also hasn't had
a professional security review or extensive real-world testing. If you plan
to run this somewhere more exposed than your home network, review the code
yourself first, particularly around file upload handling (CSV import) and
deployment configuration.
